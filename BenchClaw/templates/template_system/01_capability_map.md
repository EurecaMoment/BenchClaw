# BenchClaw 严格修正版能力维度图谱

运行层不再暴露 C1–C30 给 agent 自由选择，而收束为 8 个能力维度。原始 C 编号可作为论文分析字段保留，但 Stage4 模板选择必须使用 A 编号。

| 能力 ID | 名称 | 默认开放条件 |
|---|---|---|
| A1 | 可见对象/类别识别 | 单图、可见类别 GT |
| A2 | 计数与集合规模 | 单图、可见实例 GT；数值必须区间化 |
| A3 | 图像平面空间关系 | 单图、bbox/mask/centroid；必须 overlay 消歧 |
| A4 | 深度/远近/距离层次 | depth/3D GT 可靠；题干不得泄漏 GT 字段 |
| A5 | 可见尺度与 3D 尺寸 | 3D bbox/点云可靠；只比较可见几何 |
| A6 | 多帧时序与轨迹 | 多帧序列、帧索引、跨帧对象一致性 |
| A7 | 自我中心/姿态/3D 坐标 | agent/camera pose、heading、坐标系定义齐全 |
| A8 | 证据冲突与组合约束 | 只允许可由视觉证据直接验证的冲突/组合，不允许不可答三选 |

agent 默认只使用 `strict_core`：A1、A2、A3 和极少量 A4 稳定模板。
